# Q9. (Theory) Explain the difference between git pull, git fetch, and git clone. When would you use each? (2 marks)

### 1. git clone
•	Creates a complete local copy of a remote Git repository.
•	It is used when working with a repository for the first time.

### 2. git fetch
•	Downloads the latest changes from the remote repository without merging them into the current branch.
•	It is used when you want to review updates before applying them.

### 3. git pull
•	Downloads the latest changes from the remote repository and automatically merges them into the current branch.
•	It is used to keep your local repository up to date with the remote repository.

### Summary
•	git clone → Copy a repository for the first time.
•	git fetch → Check for new changes without merging.
•	git pull → Download and merge the latest changes.
