## Q29: Sensor Calibration in IoT Systems

### What is Sensor Calibration?

Sensor calibration is the process of **adjusting a sensor's raw output values to match
known real-world reference points**, so the readings are accurate and meaningful.

Every physical sensor has slight variations due to:
- Manufacturing differences
- Temperature and humidity effects
- Aging of components
- Environmental interference

Without calibration, two identical sensors placed in the same environment may give
different readings. Calibration corrects this error.

---

### Why is Calibration Important in IoT Systems?

In IoT, sensors are the **eyes and ears** of the system. If sensor data is wrong,
every decision the system makes is wrong.

| Without Calibration | With Calibration |
|---|---|
| Soil reads 45% moisture when actually dry | Soil correctly reads 12% — pump activates |
| Temperature shows 28°C when it is 35°C | Correct temp triggers cooling system |
| Raw value 630 — meaningless to the user | Converted to 67% — actionable data |

Real-world consequences of uncalibrated IoT sensors:
- **Smart irrigation:** overwatering or underwatering crops
- **Healthcare wearables:** wrong heart rate or SpO2 alerts
- **Industrial safety:** missing dangerous gas level thresholds

---

### How to Calibrate an Analog Soil Moisture Sensor

A soil moisture sensor outputs a raw analog value (0–1023 on Arduino).
To convert this to a percentage (0–100%), we use the
**two-point calibration method**.

#### Step 1 — Find the DRY value
- Place the sensor in completely **dry air** (no soil, no water)
- Read the raw `analogRead()` value
- This is your **maximum raw value** (sensor fully dry)
- Example: `dryValue = 850`

#### Step 2 — Find the WET value
- Place the sensor in a cup of **plain water**
- Read the raw `analogRead()` value
- This is your **minimum raw value** (sensor fully wet)
- Example: `wetValue = 300`

#### Step 3 — Map raw value to percentage
Use Arduino's `map()` function to convert any reading between
these two points into a 0–100% moisture percentage:

```cpp
int dryValue = 850;   // raw value in dry air
int wetValue  = 300;  // raw value in water

void loop() {
  int raw = analogRead(A0);

  // map raw reading to moisture percentage
  int moisturePercent = map(raw, dryValue, wetValue, 0, 100);

  // clamp to valid range (sensor can go slightly out of bounds)
  moisturePercent = constrain(moisturePercent, 0, 100);

  Serial.print("Moisture: ");
  Serial.print(moisturePercent);
  Serial.println("%");

  delay(1000);
}
```

---

### The Two-Point Calibration Method — Explained

Two-point calibration uses **exactly two known reference points**
(one at each extreme) to draw a straight line between them,
and maps all values along that line.

Raw Value

850 |  o  ← Dry point (0% moisture)

|   

|    

|     

300 |      o  ← Wet point (100% moisture)

|________________

0%         100%

Moisture %
**Formula behind it:**
Moisture% = (raw - dryValue) / (wetValue - dryValue) × 100

This is exactly what `map()` does internally in Arduino.

#### Why two points?
- **One point** only shifts the reading (offset correction)
- **Two points** correct both offset AND scale (gain correction)
- This handles the full range of the sensor accurately

#### Limitations:
- Assumes a **linear relationship** between raw value and moisture
- For higher accuracy, multi-point calibration (3+ points) can be used
- Calibration should be **redone** if sensor ages or environment changes

### Summary

| Term | Meaning |
|---|---|
| Calibration | Mapping raw sensor output to real-world values |
| Dry value | Raw reading when sensor is in dry air (0% moisture) |
| Wet value | Raw reading when sensor is in water (100% moisture) |
| Two-point method | Use two known extremes to scale all values in between |
| map() | Arduino function that does the linear scaling |
| constrain() | Clamps value to valid range to avoid negative % or >100% |
