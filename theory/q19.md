# Q19. (Theory) What is the difference between analogWrite() and analogRead() in Arduino? What is PWM and why is it used? Give a practical IoT example for each. (2 marks)

analogRead() and analogWrite() are two important Arduino functions used for handling analog signals, but they serve different purposes.

### 1. analogRead()

The analogRead() function is used to read the voltage from an analog input pin. Arduino converts the analog voltage into a digital value using its built-in 10-bit Analog-to-Digital Converter (ADC).

Syntax: analogRead(pin);
Input Pins: Analog pins (A0–A5 on Arduino Uno)
Return Value: An integer between 0 and 1023
0 V → 0
2.5 V → 512
5 V → 1023

Practical IoT Example:
In a Smart Temperature Monitoring System, a temperature sensor is connected to pin A0. Arduino continuously reads the sensor value using analogRead() and sends the data to a cloud server or mobile application for real-time monitoring.

### 2. analogWrite()

The analogWrite() function is used to generate an analog-like output using Pulse Width Modulation (PWM). It does not produce a true analog voltage but rapidly switches the output between HIGH and LOW.

Syntax: analogWrite(pin, value);
Output Pins: PWM pins (~3, 5, 6, 9, 10, 11 on Arduino Uno)
Value Range: 0 to 255
0 → Always OFF
255 → Always ON
What is PWM?

PWM (Pulse Width Modulation) is a technique in which a digital signal is switched ON and OFF rapidly. By changing the amount of time the signal remains HIGH (called the duty cycle), the average voltage supplied to the device changes.

For example:

0% Duty Cycle → 0 V (Always LOW)
50% Duty Cycle → Average voltage ≈ 2.5 V
100% Duty Cycle → 5 V (Always HIGH)

Since the switching is very fast, devices such as LEDs and motors respond as if they are receiving a variable analog voltage.

### Why is PWM Used?

PWM is used to control the power delivered to electronic devices without generating a true analog voltage. It is efficient, easy to implement, and allows smooth control of:

LED brightness
DC motor speed
Fan speed
Buzzers
Heating elements

### Practical IoT Example of PWM

In a Smart Street Lighting System, an LDR (Light Dependent Resistor) measures the surrounding light using analogRead(). Based on the light intensity, Arduino adjusts the brightness of the street LEDs using analogWrite(). During the daytime, the LEDs remain OFF, while at night their brightness increases automatically, saving energy.

### Difference Between analogRead() and analogWrite()
| **Feature**              | **`analogRead()`**                             | **`analogWrite()`**                          |
| ------------------------ | ---------------------------------------------- | -------------------------------------------- |
| **Purpose**              | Reads analog voltage from sensors              | Generates PWM output (analog-like signal)    |
| **Operation Type**       | Input operation                                | Output operation                             |
| **Pins Used**            | Analog input pins (A0–A5)                      | PWM-enabled pins (~3, 5, 6, 9, 10, 11)       |
| **Value Range**          | Returns values from **0 to 1023** (10-bit ADC) | Accepts values from **0 to 255** (8-bit PWM) |
| **Typical Applications** | LDRs, potentiometers, temperature sensors      | LEDs, motors, fans, buzzers                  |

### Conclusion:
analogRead() is used to read analog signals from sensors, while analogWrite() is used to control the output power of devices using PWM. PWM enables Arduino to simulate analog output efficiently, making it ideal for many embedded and IoT applications.
