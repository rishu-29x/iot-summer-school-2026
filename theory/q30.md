# Q30. (Theory) Explain I2C protocol with respect to IoT sensor communication. What are SDA and SCL lines? How does I2C addressing work? Name 3 common IoT sensors that use I2C. (2 marks)

I2C (Inter-Integrated Circuit) is a synchronous serial communication protocol used for communication between a microcontroller (master) and multiple peripheral devices (slaves), such as sensors, displays, and memory modules.
In IoT systems, I2C is widely used because it requires only two communication wires, making the circuit simple and reducing the number of pins required.

## How I2C Works

I2C follows a Master-Slave architecture.

Master Device: Initiates communication and controls the clock signal (e.g., Arduino, ESP32).
Slave Device: Responds only when its unique address is called (e.g., sensors).
Multiple slave devices can be connected to the same two communication lines.
SDA and SCL Lines

### I2C uses only two wires:

SDA (Serial Data Line)
Used to transfer data between the master and slave devices.
It is a bidirectional line, meaning data can flow in both directions.
SCL (Serial Clock Line)
Carries the clock signal generated by the master.
Synchronizes the data transfer between devices.

Both SDA and SCL require pull-up resistors because they use open-drain/open-collector outputs.

## How I2C Addressing Works

Each I2C slave device has a unique 7-bit (or sometimes 10-bit) address.

The communication process is as follows:

The master sends a START condition.
The master transmits the address of the slave device.
Every slave checks the transmitted address.
Only the slave whose address matches responds with an ACK (Acknowledgement) signal.
The master then reads data from or writes data to that slave.
After communication is complete, the master sends a STOP condition.

For example:

Temperature Sensor Address → 0x48
OLED Display Address → 0x3C
Pressure Sensor Address → 0x76

Since each device has a different address, many sensors can share the same SDA and SCL lines without interference.

Advantages of I2C in IoT
Requires only two communication wires.
Supports multiple devices on the same bus.
Saves microcontroller pins.
Easy to expand by adding more sensors.
Suitable for low-speed sensor communication in IoT systems.
Common IoT Sensors That Use I2C

## Some commonly used I2C sensors are:

BMP280 – Measures temperature and atmospheric pressure.
MPU6050 – Measures acceleration and angular velocity (accelerometer + gyroscope).
BH1750 – Measures ambient light intensity.

Other popular I2C devices include BME280 (temperature, humidity, pressure), OLED displays, and RTC modules.

## Practical IoT Example

Consider a Weather Monitoring Station.

An Arduino or ESP32 acts as the master.
A BMP280 sensor measures temperature and pressure.
A BH1750 sensor measures light intensity.
An OLED display shows the measured values.

All these devices communicate over the same SDA and SCL lines using different I2C addresses, reducing wiring complexity and saving GPIO pins.
